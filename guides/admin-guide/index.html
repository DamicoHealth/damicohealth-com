Admin vs Standard device roles

Every device in your organization is one of two roles:

  • Admin — can edit presets, manage cloud sync, run backups, generate donor reports, and change the subscription tier. Typically the team lead's device or the backend Mac.
  • Standard — can create encounters, view records, and run analytics. Cannot change configuration or the formulary. Can be promoted to admin by an existing admin.

The first device that creates the organization automatically becomes the admin. Additional admin devices are promoted from Admin → Devices → [device name] → Promote to Admin.

Setting an admin password

The admin password protects the Options screen and the Admin configuration sections. Without it, even an admin device can't change presets.

  1. Open Admin → Security
  2. Tap Set Admin Password
  3. Enter and confirm your password
  4. Save
Important

Forgotten admin passwords can only be reset by another admin device on the same organization. If every admin device loses the password, contact support@damicohealth.org.

Customizing presets

Presets live in the Options tab. They control what shows up as one-tap buttons everywhere else in the app. Investing 30 minutes in good presets pays off across hundreds of patient encounters.

Sites

Each clinic location your team works at. Add a name, optional location notes, and an active/inactive toggle. Sites appear as a picker on every encounter form.

Physicians

The clinicians on your team. Add name, role (MD, NP, PA, RN, etc.), and any credentials you want printed on records or reports.

Formulary (medications)

Your medication list. For each medication, capture:

  • Drug name (generic and brand)
  • Available forms (tablet, suspension, injection)
  • Strengths in stock
  • Default dosing notes (adult and pediatric)
  • Common indications

The formulary feeds the medication builder during encounters.

Procedures

The procedures your team performs. Examples: incision and drainage, suturing, dressing change, foreign body removal. Each procedure can have a default note template.

Chief complaints

Common presenting complaints — fever, abdominal pain, cough, rash. Used as one-tap buttons on the encounter form. Add ones specific to your program (e.g. snake bite, road traffic injury).

Diagnosis presets

The diagnoses your team commonly makes. For each preset:

  • Diagnosis name
  • Optional ICD-10 code
  • Default linked medications (so picking the diagnosis can suggest typical Rx)

Lab tests (custom POS/NEG or numeric)

Add the labs your program runs. For each:

  • Test name
  • Type — POS/NEG, numeric, or urinalysis dipstick
  • For numeric: units, reference range, and out-of-range flagging thresholds

Rx presets (prescription bundles)

This is one of the biggest time savers. An Rx preset bundles a full prescription — drug, dose, route, frequency, duration — into one tap. Examples:

  • Malaria — adult → artemether-lumefantrine 80/480, PO, BID, 3 days
  • UTI — adult → nitrofurantoin 100mg, PO, BID, 5 days
  • Strep throat — pediatric → amoxicillin 50 mg/kg/day, PO, divided BID, 10 days

Build a library of presets that match your program's most common prescriptions. The builder still lets you adjust on a per-patient basis.

Referral types

The kinds of referrals your team makes. Examples: surgery day, hospital transfer, follow-up clinic, OB/GYN, dental, ophthalmology. Each type can have a default destination.

Show/hide presets without deleting

Most preset editors have a Visible toggle. Turn it off to hide a preset from the encounter form without losing it — useful for seasonal items (e.g. specific antimalarials only relevant in certain trips) or items you might want to bring back.

Tip

Avoid deleting presets that have already been used in encounters. Hiding preserves historical records that reference the preset by name. Deleting may leave older encounters without a clean linked label.

Cloud Connection setup

If you're going to sync across multiple devices, you need a cloud connection. From Admin → Cloud Connection:

  1. Tap Connect
  2. Pick the cloud provider (Apple CloudKit by default; Supabase available for self-hosted organizations)
  3. Authenticate or paste your project credentials (for self-hosted Supabase)
  4. Confirm — the app verifies the connection and runs an initial schema check

Once connected, the screen shows your organization code, the sync status (last successful sync, last attempted sync, any errors), and a list of devices currently joined.

Sharing your Organization Code with team members

Each new device that joins your team needs your Organization Code. From Admin → Cloud Connection:

  1. Tap Share Organization Code
  2. Send the code through any channel (email, text, WhatsApp, or read it out loud)
  3. The teammate enters it on the Join Organization screen during their setup wizard
Note

The Organization Code is the only thing standing between your data and a stranger. Treat it like a password — share only with team members.

Manual sync workflow (sync at end of day)

The app syncs automatically when internet is available, but most field teams prefer to control when sync happens — usually at the end of each clinic day when you're back at the lodge with reliable wifi.

  1. From Admin → Sync, set sync mode to Manual
  2. Each device works offline during the day
  3. At end of day, gather devices in one place with internet
  4. On each device, tap Sync Now
  5. The backend Mac (or admin device) merges everything and pushes to the cloud
  6. Other devices pull the merged result

The sync log on the Cloud Connection screen shows progress, conflicts, and any errors.

Database backup

Even with cloud sync, run a local backup at least once per trip. From Admin → Backup:

  1. Tap Export Backup
  2. Pick a destination (Files app, AirDrop to a teammate's device, or a USB drive on Mac)
  3. The app writes an encrypted backup file containing all encounters, presets, and configuration

Restore via Admin → Backup → Import Backup on a fresh device. You'll need the same admin password to decrypt.

Generating donor reports

Donors typically want a one-page summary of activity. From Analytics → Donor Report:

  1. Pick a date range (last trip, calendar year, custom)
  2. Pick a site filter if needed (or leave empty for all)
  3. Choose the sections to include — patient counts, age and sex breakdown, top diagnoses, lab volume, surgeries, referrals fulfilled
  4. Tap Generate Report
  5. Export as PDF or CSV
Tip

Save your most-used report configurations as templates. The next trip you can re-run the same report with one tap.

Managing the subscription tier

Subscriptions are managed through Apple's App Store, but the entry point is inside the app:

  1. Open Admin → Subscription
  2. You'll see your current tier and current device count / patient count vs limits
  3. Tap a tier card to upgrade or downgrade
  4. Apple's StoreKit handles the purchase
  5. The app picks up the new entitlement immediately

To cancel or change billing, go to your iPhone or iPad Settings → [Your Name] → Subscriptions, or on Mac, App Store → [Your Name] → Account Settings → Subscriptions. Apple handles the billing — Damico Health Inc. never sees your payment details.

Note

If you cancel a paid tier, your data is preserved. The app drops you back to the Free tier, which may impose lower device or patient limits. Existing data above those limits stays accessible — you just can't add new beyond the cap until you re-subscribe.
